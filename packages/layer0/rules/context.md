# Context Rules

1. **Read progress first.** Before any work, read `foundry-progress.json` for current state and decisions.
2. **Use codemap.** Before exploring unfamiliar code, check `.foundry/codemap.md` for structure overview.
3. **Check git log.** Review recent commits for context on what changed and why.
4. **Handoff notes.** If `.foundry/handoff-note.md` exists, read it — it contains critical context from the previous session.
5. **Compact strategically.** Use `/foundry-compact` at phase boundaries, not mid-task.
