# Scope Rules

1. **Stay in scope.** Only modify files relevant to the current task. If a plan exists, check `relevant_files`.
2. **No drive-by refactors.** Don't "improve" code outside the task boundary. File a separate task instead.
3. **No unnecessary .md creation.** Don't create documentation files unless the task explicitly requires it.
4. **Ask before expanding scope.** If a task requires changes beyond originally planned files, confirm with user first.
