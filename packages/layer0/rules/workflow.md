# Workflow Rules

1. **Plan before implementing.** Non-trivial changes require `/foundry-plan` first. No code until plan approved.
2. **Verify continuously.** After edits, check typecheck/lint/test results. Fix immediately — never defer.
3. **Track progress.** Update `foundry-progress.json` after completing any task or subtask via `/foundry-progress`.
4. **Checkpoint often.** Use `/foundry-checkpoint` after meaningful progress — verify, commit, update progress atomically.
5. **Read before writing.** Always read `foundry-progress.json` and recent git log before starting work. Use `/foundry-context` for unfamiliar areas.
6. **One task at a time.** Complete current task fully before starting next. No half-finished work.
